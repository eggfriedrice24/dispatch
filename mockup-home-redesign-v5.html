fix: resolve auth token refresh race condition

alex-k · opened 1h ago fix/auth-token-refresh → main
Review requested 6/8 checks +142 −38

Fixes a race condition where the auth token could expire mid-refresh, causing parallel API calls to fail with 401s and trigger multiple simultaneous refresh attempts.

The fix introduces a mutex around the refresh flow — concurrent callers wait for the in-flight refresh to resolve rather than each spawning their own. Also adds a tokenExpiryBuffer of 30s so we refresh proactively before the token actually expires.
